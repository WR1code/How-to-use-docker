可以从这几类来看你本地 Docker 里有什么：

## 1. 看本地有哪些镜像

```bash
docker images
```

或新版命令：

```bash
docker image ls
```

你会看到类似：

```bash
REPOSITORY      TAG       IMAGE ID       CREATED        SIZE
ros             noetic    xxxxxxxx       2 weeks ago    2.3GB
ubuntu          20.04     xxxxxxxx       1 month ago    72MB
```

镜像就是“系统模板”，比如 ROS1 环境、Ubuntu 环境。

---

## 2. 看正在运行的容器

```bash
docker ps
```

看所有容器，包括已经停止的：

```bash
docker ps -a
```

容器就是镜像启动后的“实例”。

---

## 3. 进入某个容器内部

先查容器名或 ID：

```bash
docker ps -a
```

然后进入：

```bash
docker exec -it 容器名 bash
```

例如：

```bash
docker exec -it ros1_container bash
```

如果容器里没有 `bash`，用：

```bash
docker exec -it 容器名 sh
```

进去后就像进了一个小型 Linux 系统一样：

```bash
ls
cd /
pwd
ls /root
ls /home
```

---

## 4. 如果容器没运行，先启动它

```bash
docker start 容器名
```

然后再进入：

```bash
docker exec -it 容器名 bash
```

---

## 5. 查看容器详细信息

```bash
docker inspect 容器名
```

这个可以看到挂载目录、网络、端口、启动命令等。

如果只想看挂载了哪些本地文件夹：

```bash
docker inspect 容器名 | grep Mounts -A 30
```

---

## 6. 查看 Docker 占用了多少空间

```bash
docker system df
```

更详细：

```bash
docker system df -v
```

---

## 7. 查看 Docker volume

```bash
docker volume ls
```

查看某个 volume 位置：

```bash
docker volume inspect volume名字
```

volume 一般是 Docker 持久化数据的位置。

---

## 8. 看容器日志

```bash
docker logs 容器名
```

持续查看：

```bash
docker logs -f 容器名
```

---

## 9. 看容器里运行了什么进程

```bash
docker top 容器名
```

---

## 10. 对你这种 ROS1 Docker，常用流程是

```bash
docker ps -a
```

找到 ROS 容器后：

```bash
docker start 容器名
docker exec -it 容器名 bash
```

进去之后：

```bash
source /opt/ros/noetic/setup.bash
roscore
```

如果想看你的工作空间：

```bash
ls ~/catkin_ws
ls ~/catkin_ws/src
```

如果不知道工作空间在哪，可以搜：

```bash
find / -name "catkin_ws" 2>/dev/null
```

或者搜 ROS 包：

```bash
find / -name "package.xml" 2>/dev/null
```

---

你最常用的几条其实就这几个：

```bash
docker images
docker ps -a
docker start 容器名
docker exec -it 容器名 bash
docker inspect 容器名
docker logs 容器名
```
